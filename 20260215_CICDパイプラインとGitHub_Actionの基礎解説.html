詳細要約

CI/CDパイプラインの概要

現在の開発環境として、ローカル（カーソル）→GitHub→Google Cloud（本番環境）という3層構造が存在している。CI/CDパイプラインを構築することで、これらを自動的に連携させることができる。

自動デプロイの仕組み

CI/CDパイプラインを繋ぐと、ローカルでコミット・プッシュした際に、自動的にGitHubに保存され、さらに本番環境にも自動反映される流れが構築される。通常は手動で本番環境への反映指示が必要だが、パイプライン構築により完全自動化が実現できる。

GitHub Actionの役割

GitHubに標準搭載されているGitHub Actionという機能により、本番環境へのデプロイ前に自動テストが実行される。オンライン上で動作確認を行い、問題がなければ自動的に本番環境に反映される仕組みとなっている。

実務での位置づけ

カーソルが優秀なため、現代の開発環境では必須ではないが、エンジニアとの協業時のコミュニケーションのために知識として理解しておくべき内容。特に開発したものを商品化する際にエンジニアとやり取りする場面で重要となる。
